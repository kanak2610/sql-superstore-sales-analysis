# Superstore Sales Analysis using SQL

## Overview

This project demonstrates SQL-based sales analysis on the Superstore dataset. The objective is to extract meaningful business insights using advanced SQL techniques such as Subqueries, Common Table Expressions (CTEs), Window Functions, Joins, and Aggregations.

---

## Project Objectives

* Load and transform raw sales data into structured tables.
* Perform customer and sales analysis using SQL.
* Identify top-performing and low-performing customers.
* Apply ranking techniques using Window Functions.
* Generate business insights from transactional sales data.

---

## Dataset

**Source:** Superstore Dataset

The dataset contains information related to:

* Orders
* Customers
* Products
* Sales
* Profit
* Categories and Sub-Categories

---

## Database Schema

### Tables Created

| Table Name     | Description             |
| -------------- | ----------------------- |
| superstore_raw | Raw imported dataset    |
| customers      | Customer information    |
| orders         | Order details and sales |
| products       | Product information     |

## SQL Techniques Implemented

### 1. Subqueries

* Orders with sales above average sales.
* Highest sales order for each customer.

### 2. Common Table Expressions (CTEs)

* Total sales calculation per customer.
* Customer sales aggregation.

### 3. Window Functions

* ROW_NUMBER()
* RANK()

### 4. Joins

* Customer-wise sales analysis.
* Customer ranking based on total sales.

### 5. Aggregations

* SUM()
* AVG()
* COUNT()
* GROUP BY

---

## Business Problems Solved

### Above Average Sales Analysis

Identified orders whose sales exceeded the overall average sales value.

### Highest Order per Customer

Determined the maximum-value order placed by each customer.

### Customer Revenue Analysis

Calculated total revenue generated by each customer.

### Customer Ranking

Ranked customers according to their total sales contribution.

### Single Order Customers

Identified customers who placed only one order.

### Top and Low Performing Customers

Analyzed customer segments based on revenue generation.

---

## Key Insights

* A small group of customers contributes a significant portion of total revenue.
* Customer ranking helps identify high-value customers for retention strategies.
* Several customers placed only one order, indicating opportunities for customer engagement.
* Above-average sales transactions contribute disproportionately to overall business performance.
* Sales aggregation enables data-driven customer segmentation.
